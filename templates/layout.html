Welcome to the TeamPCP Hunter

Real-time PyPI supply-chain threat detection platform

Detection Pipeline

1
📡 Monitor PyPI RSS Feed
Poll pypi.org/rss/updates.xml every 2 min. Track all newly published or updated packages in real time.
↓
2
🔗 Resolve GitHub Repository
Extract the linked GitHub repo from project_urls and home_page in PyPI metadata. Skip packages with no GitHub link.
↓
3
🔍 Verify Version on GitHub
Check if the PyPI version has a matching release or tag. Handles PEP 440 ↔ semver, monorepo prefixes, and pre-release normalization.
↓
4
🚨 Flag & Score Anomalies
Flag when PyPI has a version not on GitHub. Confidence scoring: rapid publish ⚡, high downloads 🎯, yanked versions ⚠️.
↓
5
✅ Auto Re-verify (30 min)
Re-check flagged packages after 30 min. If a matching GitHub release appears (delayed publish), the flag is automatically cleared.
🎯 Why TeamPCP? — Detects the exact attack pattern from the telnyx and litellm compromises: attackers steal PyPI API tokens and publish malicious versions directly to PyPI without ever touching GitHub. This system catches that discrepancy in real time.
