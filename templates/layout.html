Welcome to the TeamPCP Hunter

Real-time PyPI supply-chain threat detection platform

How It Works

1
Monitor PyPI Feed — We continuously poll the PyPI RSS updates feed every 2 minutes, tracking every newly published or updated package in real time.
2
Resolve GitHub Repo — For each package, we extract the linked GitHub repository from PyPI project metadata (project_urls, homepage).
3
Verify Version Match — We check if the PyPI version has a corresponding release or tag on GitHub. PEP 440 ↔ semver normalization handles format differences.
4
Detect Anomalies — Packages are flagged when PyPI publishes a version with no matching GitHub release. Risk signals (rapid publish, high downloads, yanked versions) boost confidence scoring.
5
Auto Re-verify — Flagged packages are re-checked after 30 minutes. If a matching GitHub release appears (delayed publish), the flag is automatically cleared.
🎯 Why TeamPCP? — This platform detects the exact attack pattern used by TeamPCP (telnyx, litellm compromises): attackers steal PyPI tokens and publish malicious versions directly to PyPI without touching GitHub. Our system catches this discrepancy in real time.
